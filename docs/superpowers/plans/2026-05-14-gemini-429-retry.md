# Gemini Rate Limits Fix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implementar política de WaitAndRetry (Exponential Backoff com Jitter) para resolver os erros de HTTP 429 (Too Many Requests) que causam falhas abruptas no Gemini durante processos em background, mantendo o Circuit Breaker existente.

**Architecture:** Evoluir o uso do Polly no `LLMManager.cs`. Atualmente há apenas um Circuit Breaker que falha a requisição imediatamente e abre o circuito. A solução é combinar o CircuitBreaker com um `WaitAndRetryAsync` usando `Policy.WrapAsync(cb, retry)`, e usar uma tupla para continuar verificando o estado do circuito.

**Tech Stack:** .NET 10, ASP.NET Core, C#, Polly

- [ ] **Step 2: Substituir GetCircuitBreaker por GetResiliencePolicy**
Mudar a função para construir o wrap e retornar a tupla. O CircuitBreaker fica externo, e o Retry fica interno.

```csharp
    private (Polly.Wrap.AsyncPolicyWrap Policy, Polly.CircuitBreaker.AsyncCircuitBreakerPolicy CircuitBreaker) GetResiliencePolicy(string providerName)
    {
        return _resiliencePolicies.GetOrAdd(providerName, _ =>
        {
            var cb = Polly.Policy
                .Handle<Exception>(ex => 
                    ex.Message.Contains("429") || 
                    ex.Message.Contains("timeout") || 
                    ex is HttpRequestException || 
                    ex is TimeoutException)
                .CircuitBreakerAsync(
                    exceptionsAllowedBeforeBreaking: 3,
                    durationOfBreak: TimeSpan.FromSeconds(30),
                    onBreak: (ex, breakDelay) =>
                    {
                        _logger.LogWarning(ex, "🚨 Circuit Breaker for {Provider} tripped! Breaking for {Delay}s", providerName, breakDelay.TotalSeconds);
                    },
                    onReset: () =>
                    {
                        _logger.LogInformation("✅ Circuit Breaker for {Provider} reset.", providerName);
                    },
                    onHalfOpen: () =>
                    {
                        _logger.LogInformation("⏳ Circuit Breaker for {Provider} is half-open (testing next request).", providerName);
                    });

            var retry = Polly.Policy
                .Handle<Exception>(ex => 
                    ex.Message.Contains("429") || 
                    ex.Message.Contains("timeout") || 
                    ex is HttpRequestException || 
                    ex is TimeoutException)
                .WaitAndRetryAsync(
                    retryCount: 4,
                    sleepDurationProvider: retryAttempt => 
                        TimeSpan.FromSeconds(Math.Pow(2, retryAttempt)) + TimeSpan.FromMilliseconds(Random.Shared.Next(0, 1000)),
                    onRetry: (ex, time, attempt, context) =>
                    {
                        _logger.LogWarning(ex, "⏳ Rate limit or timeout on {Provider}. Retrying in {Time}s (Attempt {Attempt}/4)", providerName, time.TotalSeconds, attempt);
                    });

            return (Polly.Policy.WrapAsync(cb, retry), cb);
        });
    }
```
